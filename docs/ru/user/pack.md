# Упаковка

Pack-операции пересобирают firmware-образы из содержимого workspace. Это
обратные операции к соответствующим unpack-операциям.

## PACK_FILESYSTEM_IMAGE

Вход: дерево директорий в workspace.

Обязательные инструменты зависят от обнаруженной filesystem:
- EROFS → `mkfs.erofs`
- ext4 → `mke2fs` + `e2fsdroid`

Результат: один файл `<name>.img` в exports-директории workspace.

Packer:
1. Обходит входную директорию.
2. Строит filesystem-образ с корректным ownership, permissions и
   SELinux-контекстами (читает `file_contexts`, если он есть).
3. Считает SHA-256 результата для проверки.

## PACK_BOOT_IMAGE

Вход: `kernel`, `ramdisk.cpio`, `dtb`, информация о заголовке (из
`UNPACK_BOOT_IMAGE`).

Обязательные инструменты: `magiskboot`, `mkbootimg`.

Результат: новый `boot.img`.

Packer:
1. Пересобирает boot image, используя исходную версию заголовка.
2. Пересчитывает поля заголовка (size, CRC и т.д.).
3. Если доступен `magiskboot`, переподписывает образ для Magisk-совместимости.

## PACK_SUPER

Вход: набор partition-образов (`system.img`, `vendor.img`, ...).

Обязательные инструменты: `lpmake`, `img2simg` (если нужен sparse output).

Результат: один `super.img` со всеми logical partitions.

Packer использует метаданные исходного `super.img` (размеры разделов, группы,
флаги) и подставляет новые partition-образы. Если новые образы больше
оригиналов, pack провалится — нужно их уменьшить или увеличить super-раздел.

## Профиль безопасности

Все pack-операции не-деструктивны для устройства:
- `isDestructive = false` (меняются только workspace-файлы)
- `requiresRoot = false`
- `confirmationLevel = STANDARD` (yes/no перед запуском)
- `supportsDryRun = true`

Pack-операции **НЕ** касаются устройства — они только создают новые образы
в workspace. Чтобы поместить новые образы на устройство, нужно явно
экспортировать их и запустить [FLASH_PREPARE](flash-safety.md).

## Готовность toolchain

Перед любой pack-операцией `OperationPlanner` проверяет план toolchain и
отказывается запускаться, если какой-то обязательный инструмент отсутствует.
On-screen список блокировок точно говорит, чего не хватает. Используйте
`ToolchainInstaller` (запускается автоматически при старте приложения или
вручную из настроек), чтобы извлечь недостающие бинарники по манифесту.

## Экспорт

После успешной pack-операции используйте кнопку **Export**, чтобы записать
результат в видимое пользователю место. Exporter поддерживает три политики:
- `AskPerFile` — выбирать назначение для каждого файла.
- `TreeFolder` — выбрать папку один раз, все файлы туда.
- `MediaStoreExport` — сохранить в `Download/MIO-KITCHEN/`.

SHA-256 считается во время экспорта, чтобы вы могли проверить результат.
