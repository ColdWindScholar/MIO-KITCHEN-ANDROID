# Безопасность flash

**Прочитайте эту страницу целиком перед запуском любой flash-операции.**

Flash-операции — единственные деструктивные операции в MIO-KITCHEN. Они
записывают firmware-образы в разделы вашего устройства — ошибка может
окирпичить устройство.

## Что делает flash

`FLASH_PREPARE` — единственная типизированная flash-операция в
`FirmwareOperation`. Она **НЕ** прошивает — она создаёт готовый к запуску
flash-скрипт, который можно просмотреть перед выполнением.

Реальная прошивка делается вами вручную через `fastboot` или `dd` из
recovery-shell. MIO-KITCHEN никогда не пишет в разделы автоматически.

## Профиль безопасности

```text
isDestructive           = true
requiresBackup          = true
requiresRoot            = true
requiresDeviceConnection= true
supportsDryRun          = true
confirmationLevel       = DESTRUCTIVE
```

Поскольку `confirmationLevel = DESTRUCTIVE`, UI требует, чтобы вы ввели
`FLASH` (или эквивалент) в текстовое поле перед запуском операции.
Случайный one-tap flash исключён.

## Предстартовый чек-лист

Перед запуском `FLASH_PREPARE` проверьте:

1. **Устройство подключено** через ADB или fastboot. Флаг
   `requiresDeviceConnection` соблюдается — executor откажется запускаться
   иначе.
2. **Root доступен**. Flash без root невозможен; executor заблокирует план,
   если `device.hasRoot != true`.
3. **Backup существует**. Сделайте полный backup текущей прошивки через `dd`
   или TWRP перед прошивкой чего-либо.
4. **Батарея заряжена** минимум до 50%. Потеря питания во время flash
   окирпичивает устройство.
5. **Bootloader разблокирован** (для `boot` / `vbmeta` разделов). AVB
   отклонит подписанные образы на заблокированном bootloader.
6. **Образ подходит вашему устройству**. Прошивка `system.img` от другой
   модели окирпичит его.

## Сначала dry-run

Всегда запускайте `FLASH_PREPARE` с `dryRun = true` сначала. Executor
использует `DryRunShellRuntime`, который печатает запланированные команды
без их выполнения. Просмотрите напечатанные команды — они должны совпадать
с тем, что вы ввели бы вручную.

## Flash-скрипт

Flash-скрипт — это shell-скрипт (`flash.sh`), который содержит:
- `fastboot flash` команды для каждого раздела.
- `dd` fallback-и для разделов, которые нельзя прошить через fastboot.
- Финальный `fastboot reboot` (закомментирован по умолчанию).

Скрипт пишется в exports-директорию workspace. **Просмотрите его построчно
перед запуском.**

## Что делать, если flash провалился

1. **Не перезагружайтесь.** Провалившийся flash часто оставляет устройство в
   промежуточном состоянии, которое можно восстановить.
2. **Перепрошивайте провалившийся раздел** исходным образом из backup.
3. **Если устройство в boot-loop**, загрузитесь в recovery (TWRP) и
   восстановите полный backup.
4. **Если устройство совсем не отвечает**, может понадобиться EDL-режим
   (Qualcomm) или BROM-режим (MediaTek) для восстановления — за рамками
   MIO-KITCHEN.

## Что MIO-KITCHEN НЕ делает

- Не auto-flashит. Нет кнопки "Flash now".
- Не определяет, подходит ли образ вашему устройству. Вы должны проверить.
- Не меняет состояние блокировки bootloader. Используйте
  `fastboot oem unlock` (или vendor-specific эквивалент) вручную.
- Не делает backup текущей прошивки. Сначала используйте `dd` или TWRP.
